chmod

파일의 퍼미션을 변경하기 위한 명령어

 # ls -l lin.txt

 --w-------. joon ihd 513 Dec 22 21:05 lin.txt

 # chmod 775 lin.txt

 -rwxrwxr-x. joon ihd 513 Dec 22 21:05 lin.txt

chown

파일의 소유자나 소유 그룹을 변경하기 위한 명령어

사용자 변경

chown 사용자명 대상

그룹 변경

chown :그룹 대상

사용자 + 그룹 변경

chown 사용자명:그룹 대상

 # ls -l lin.txt

 --w-------. joon ihd 513 Dec 22 21:05 lin.txt

 # chown :kait lin.txt

 # ls -l lin.txt

 -rwxrwxrwx. joon kait 513 Dec 22 21:05 lin.txt

chgrp

파일의 사용자 그룹을 변경하기 위한 명령어

 # ls -l lin.txt

 --w-------. joon ihd 513 Dec 22 21:05 lin.txt

 # chgrp kait lin.txt

 # ls -l lin.txt

 -rwxrwxrwx. joon kait 513 Dec 22 21:05 lin.txt

tar

tar는 리눅스 파일시스템을 압축하는 명령어

tar -cvf [파일명.tar] [파일명] : .tar로 압축

tar -xvf [파일명.tar] : .tar 압축 해제

tar -zcvf [파일명.tar.gz] [폴더명] : .tar.gz로 압축

tar -zxvf [파일명.tar.gz] : .tar.gz 압축 해제

tar -rvf [파일명.tar] [파일명] : 기존 압축에 새파일 추가

tar -tvf [파일명.tar] : 내용확인

crontab

시스템 구축 혹은 데이터 구축에서 확인할 때 주기적으로 어떤 일을 해야 할 경우 특정 시간에 어떤 일을 할때 사용 하는 명령어

  • crontab -u 사용자

    • 특정 사용자의 crontab 작성

  • crontab -e

    • 예약된 작업 수정(만들기 가능)

  • crontab -l

    • 예약된 작업 리스트

  • crontab -r

    • 예약된 작업 삭제

  • crontab -d

    • 표준 출력으로 크론탭 내용 출력

emacs

리처드 스톨만이 매크로 기능이 있는 텍스트 교정 및 편집기로 개발하였다. 이후 제임스 고슬링이 LISP 언어에 의한 환경 설정 및 에디터의 기능을 확장 시킬 수 있는 기능을 포함하여 배포하였다. 기능이 너무 많아 대응하는 LISP 코드를 불러 오는데에 오랜 시간이 걸리고 초보자가 사용하기 힘들다.

pico

워싱턴 대학에서 만든 유닉스용 편집기로 윈도우의 메모장처럼 간단하게 사용하도록 만들었다. 리눅스 초기 배포판에는 포함 되었으나, 현재 대부분의 배포판에는 제공되지 않는다.

vi editor

emacs 및 pico와 유사하게 셸 명령행에서 'vi [파일명]' 이라 입력

vi는 다른 편집기들과 다르게 모드(mode)형 편집기

  • 명령모드

    • 커서의 이동, 수정, 삭제, 복사, 붙이기, 탐색

    입력모드에서 [Esc]키를 누르면 명령모드로 전환

  • 입력모드

    • 명령모드에서 i, a, o, I, A, O 등을 누르면 입력모드로 전환

    글자를 입력하는 문서를 만드는 모드

    화면 아래에 --INSERT--라 표시

  • 콜론모드

    • 명령모드에서 ':' 키를 입력했을 때 화면 맨 아랫줄에서 명령을 수행하는 모드

    저장, 종료, 탐색, 치환 및 vi 환경설정

gedit

GNOME 기반의 문서 편집기로 용량은 작지만 다양하고 강력한 기능을 제공한다.

임베디드 시스템

마이크로컨트롤러등을 내장하여 특정한 기능을 반복적으로 수행하기 위해 하드웨어와 소프트웨어를 결합하여 만든 전자제어시스템으로 규모가 작고 단순한 시스템에 사용되며 보통 운영체제를 포함하지 않는다.

라즈베리파이

영국에서 교육용으로 개발된 소형 싱글모드 컴퓨터이다. 저렴한 가격에 그래픽성능이 뛰어나고 Wi-Fi, Bluetooth를 지원하고 다양한 센서 연결을 통한 사물인터넷 환겨 구축이 용이하다. 기본 운영체제로 리눅스가 사용되고 있다.

아두이노

2005년 이탈리아에서 하드웨어에 익숙하지 않는 학생들을 위해 개발되었다. 오픈소스기반 단일보드 마이크로컨트롤러로 다양한 센서를 연결하여 데이터를 전송 할 수 있어 임베디드 시스템 환경 개발이 용이하다. 운영체제로 Linux를 지원한다.

ICANN

1998년에 설립된 기관으로 인터넷, 도메인 이름, IP 주소, 프로토콜의 범주와 포트번호 할당, DNS관련 기능 감독, 도메인 분쟁의 조율과 중재 등을 담당한다.

ANSI

미국의 산업 표준을 제정하는 기구이다. 이 기관에서 제정한 것중 대표적으로는 ASCII 코드가 있다.

RFC

인터넷에서 기술을 구현하는 데에 필요한 절차 등을 제공하는 공문서 간행물이다. 이 문서 내용의 대부분은 인터넷상에서 기술을 구현함에 있어서 요구되는 상세한 절차와 기본 툴을 제공한다.

LAN 구성 방식

  • 링(Ring) 형

    • 모든 컴퓨터가 논리적인 원을 이루면서 연결
    데이터는 원형으로 각각의 컴퓨터를 거쳐서 이동

  • 망(Mesh) 형

    • 각 네트워크 장비가 여러 개의 인터페이스를 갖추고 상호간 연결하는 방식
    설치 비용이 많이 드는 관계로 LAN을 구성할 때보다 라우터를 이용해서 LAN과 LAN을 연결하거나 백본망을 구성할 때 주로 사용된다.
    장애 발생 시에도 다른 시스템의 영향이 적고 우회할 수 있는 방법이 존재하여 신뢰성이 높다.

  • 버스(Bus) 형

    • 하나의 통신회선에 여러 컴퓨터를 연결해서 전송하는 방법
    연결된 컴퓨터 수에 따라 네트워크 성능이 변동
    단말기의 추가 및 제거가 용이, 설치비용 저렴

  • 스타(Star) 형

    • 중앙의 제어기를 중심으로 모든 기기는 Point-to-Point 방식으로 연결
    고속의 대규모 네트워크에 이용하고 일부 장애가 발생해도 전체 네트워크에 영향이 없다.
    설치 비용이 높다

OSI 7계층

  • 1 계층 : 물리 계층(Physical Layer)

    • 전송 매체에 대한 연결 및 종료를 위한 기계적, 절차적 특성에 관한 세부사항을 정의하고, 통신채널을 통해 전송되는 사용자 장치의 디지털 데이터를 이에 상응하는 신호 변환 등을 담당한다.
    허브, 리피터

  • 2 계층 : 데이터 링크 계층(Data Link Layer)

    • MAC 주소, 전송제어, 오류 제어, 전송 단위의 순서 제어, 흐름 제어
    브리지, 스위치

  • 3 계층 : 네트워크 계층(Network Layer)

    • 데이터 전송과 경로 선택에 관한 서비스를 제공한다. 혼잡제어, 패킷의 분할과 병합, 인터네트워킹 등이 있다.
    라우터

  • 4 계층 : 전송 계층(Transprot Layer)

  • 5 계층 : 세션 계층(Session Layer)

    • 응용 프로그램 사이의 접속 설정 및 유지, 종료에 관한 세션 관리 기능, 데이터의 전송순서 및 동기점의 위치를 제공
    데이터 재전송, 데이터 복구

  • 6 계층 : 표현 계층(Presentation Layer)

    • 송신자와 수신자가 서로 다른 표현 방식을 사용하는 경우 코드와 문자 등을 변역하여 일관되게 전송 데이터를 서로 이해할 수 있도록 하는 기능 제공
    인코딩, 암호화와 해독, 압축 및 압축 해제

  • 7 계층 : 응용 계층(Application Layer)

    • 응용 프로그램과 연계하여 사용자에게 편리한 관경을 제공
    전자우편 송수신, 웹 서버에 접속, 파일 전송, 원격접속

계층 이름 전송 단위 역활
7 계층 응용 계층 data 사용자에게 직접 제공하는 서비스 및 응용프로그램 간의 정보교환
6 계층 표현 계층 data 송수신할 정보의 표현에 차이가 있는 응용 프로그램들에게 연결 제공
5 계층 세션 계층 data 응용 프로그램 간의 통신을 관리하기 위한 방법과 동기화를 유지하는 서비스 제공
4 계층 전송 계층 segment 정보의 신뢰성 있고 투명한 데이터 전송에 관한 서비스 제공
3 계층 네트워크 계층 packet 정보의 전송을 위한 경로 선택
2 계층 데이터 링크 계층 frame 물리적인 링크를 통하여 신뢰성 있는 정보의 전송을 위한 통신로 설정
